# JobTracker — Командный центр соискателя

Полноценный MVP для управления поиском работы: источники вакансий, канбан-воронка, лента активности, контакты, геймификация, образование, клиентский сейф кредов, дашборд и аналитика.

## Стек

- Frontend: React + TypeScript + Vite + Tailwind CSS, `@dnd-kit/core`, Recharts.
- Backend: FastAPI + SQLAlchemy + JWT.
- БД: PostgreSQL в Docker Compose. Для локального запуска без Docker backend автоматически использует SQLite, если `DATABASE_URL` не задан.
- Шифрование кредов: Web Crypto API, AES-GCM, PBKDF2. Мастер-пароль не отправляется на сервер.

## Быстрый запуск

```bash
docker-compose up --build
```

Откройте:

- UI: <http://localhost:5173>
- API: <http://localhost:8000/docs>

## Первый вход

1. Нажмите «Создать аккаунт».
2. Укажите email, имя и пароль.
3. После регистрации backend создаст стандартные колонки канбана:
   - На рассмотрении
   - Отклик отправлен
   - Просмотрен
   - Тестовое задание
   - Собеседование
   - Оффер
   - Отказ / Архив
4. Создайте источник в разделе «Источники».
5. Добавляйте вакансии кнопкой «+ Новая вакансия».

## Сейф кредов и мастер-пароль

1. Откройте раздел «Сейф».
2. Введите мастер-пароль. При первом использовании он фактически устанавливается локально на время сессии.
3. Добавьте запись: источник, логин, пароль и секретные заметки.
4. Браузер шифрует данные через AES-GCM. В API и БД уходит только blob вида `salt + iv + ciphertext`.
5. Если ввести другой мастер-пароль, расшифровка старых записей не удастся. Сервер не может восстановить мастер-пароль.

## Быстрое добавление вакансий без расширения

Расширение браузера не входит в MVP. Используйте ручную кнопку «+ Новая вакансия».

Дополнительно можно создать bookmarklet:

1. Создайте новую закладку в браузере.
2. В поле URL вставьте:

```javascript
javascript:(()=>{const title=document.title;const url=location.href;alert('Откройте JobTracker и нажмите + Новая вакансия. Заголовок: '+title+' URL: '+url)})()
```

В будущей версии bookmarklet/расширение можно доработать, чтобы оно открывало форму JobTracker с prefill-параметрами.

## Основные API

- `POST /api/auth/register`, `POST /api/auth/login` — JWT-аутентификация.
- `GET/POST/PUT/DELETE /api/sources` — источники вакансий.
- `GET/POST/PUT/DELETE /api/vacancies` — вакансии.
- `POST /api/vacancies/{id}/move` — перемещение карточки и начисление XP.
- `GET/POST /api/contacts` — контакты.
- `GET /api/activities` — хронологическая лента.
- `GET/POST /api/events` — календарные события.
- `GET/POST/PUT /api/materials`, `/api/courses`, `/api/roadmap` — образование.
- `GET/POST/PUT/DELETE /api/credentials` — зашифрованные креды.
- `GET /api/analytics`, `GET /api/export.csv` — аналитика и CSV.

## Переключение SQLite/PostgreSQL

Backend читает `DATABASE_URL`.

- По умолчанию: `sqlite:///./jobtracker.db`
- PostgreSQL: `postgresql+psycopg2://jobtracker:jobtracker@db:5432/jobtracker`

## Примечания по MVP

- Метаданные по ссылке в форме вакансии оставлены как ручное поле описания, чтобы избежать CORS/скрейпинг-ограничений на клиенте.
- Сертификаты и файлы можно загружать через `/api/upload`; URL хранится в соответствующих сущностях.
- Лидерборд реализован локально через личные XP, уровни и достижения.
- Ежедневные челленджи генерируются по UTC при обращении к дашборду.
